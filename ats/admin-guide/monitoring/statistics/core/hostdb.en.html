HostDB¶

The HostDB component of Traffic Server manages origin server name resolution and host health checks. The statistics documented here can help to ensure that your Traffic Server instances are not spending an unreasonable amount of timing resolving your origin servers’ hostnames prior to object revalidation or retrieval.

counter

Represents the total number of origin server name resolutions which were satisfied by entries in the HostDB lookup cache, since statistics collection began.

proxy.process.hostdb.total_serve_stale¶
Collection
global
Type
counter
Datatype
integer

Represents the total number of origin server name resolutions which were satisfied by entries in the HostDB lookup cache while those entries were stale, since statistics collection began. See proxy.config.hostdb.serve_stale_for for how this feature is configured.

proxy.process.hostdb.total_lookups¶
Collection
global
Type
counter
Datatype
integer

Represents the total number of origin server name resolutions which have been performed, since statistics collection began, regardless of whether they were satisfied by the HostDB lookup cache or required DNS lookups.
